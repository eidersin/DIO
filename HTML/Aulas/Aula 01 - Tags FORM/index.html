Módulo 2 - Trabalhando com Formulários

Índice

  • Diferença entre GET e POST

  1. Método GET:

    • Utilizado para solicitar dados do servidor.
    • Os parâmetros são anexados à URL na forma de uma string de consulta (query string).
    • Limitado em termos de quantidade de dados que pode enviar.
    • Os dados são visíveis na URL, o que pode representar um problema de segurança ao lidar com informações sensíveis.
      •         Exemplo:
        <form action="/exemplo" method="GET">
        <label for="nome">Nome:</label>
        <input type="text" id="nome" name="nome">
        <input type="submit" value="Enviar">
        </form>
        
      
        
        
        
        A URL resultante pode ser algo como /exemplo?nome=valorDigitado.

  2. Método POST:

    • Utilizado para enviar dados ao servidor de forma mais segura e eficiente.
    • Os parâmetros são enviados no corpo da requisição HTTP, não na URL.
    • Pode enviar uma quantidade maior de dados em comparação com o método GET.
    • Mais apropriado para o envio de dados sensíveis, como senhas.
      •         Exemplo:
        <form action="/exemplo" method="POST">
        <label for="nome">Nome:</label>
        <input type="text" id="nome" name="nome">
        <input type="submit" value="Enviar">
        </form>
        
      
        
        
        
        Os dados são enviados no corpo da requisição, não visíveis na URL.
      Em resumo, o método GET é adequado para solicitar dados do servidor e tem suas limitações em termos de quantidade e segurança dos dados. Já o método POST é mais apropriado para enviar dados ao servidor, especialmente dados sensíveis, e permite o envio de uma quantidade maior de informações.
